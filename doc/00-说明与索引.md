# 通用文档模板（索引）

本目录提供可复用模板，覆盖 A/B/C/D 四个窗口的阶段任务、执行规范、方案、UI/接口、需求、测试发布、运维可观测性与决策记录等。
模板遵循现有文档风格：编号章节、`---` 分隔、硬性规则用 **必须/写死/固定** 标注。
内容已做"细节版"骨架，适配不同项目，不绑定任何具体技术或产品。

---

## 0. 重要说明（必读）

⚠️ **本目录是模板体系，不是已填充的项目文档！**

- 所有 `...` 和 `<占位符>` 需要替换为实际内容
- 在开始项目前，MAIN 必须先填充以下核心文档：
  1. `doc/15-需求规格与验收指标模板.md` → 填写需求
  2. `doc/14-执行计划模板.md` → 填写里程碑和任务
  3. `doc/09-接口规范模板.md` → 填写接口定义（如有）
- 填充后的文档才能作为 SSOT 来源
- 子 AI 发现文档未填充时，应提问 MAIN 而非自行假设

---

## 1. 使用方式
- 复制模板到目标路径
- 将 `<占位符>` 替换为实际内容
- 保留编号与结构
- 落地后在文件末尾追加变更记录

---

## 2. 命名与编号约定（统一）
- 窗口仅允许：A / B / C / D
- 任务 ID：`S<阶段>-A-<序号>` / `S<阶段>-B-<序号>` / `S<阶段>-C-<序号>` / `S<阶段>-D-<序号>`
- 需求 ID：`R<阶段>-<序号>`
- 决策 ID：`DR-YYYYMMDD-<序号>`
- 状态枚举：TODO / DOING / DONE / BLOCKED
- 优先级枚举：P0 / P1 / P2
- 进度记录格式：`Txx [TODO|DOING|DONE|BLOCKED] owner=A|B|C|D date=YYYY-MM-DD note=...`
